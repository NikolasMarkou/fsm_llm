# deepagents — Reverse-Engineering & Absorption Report

> Audience: fsm_llm maintainers.
> Source under review: `langchain-ai/deepagents` v0.5.4 (pyproject `2026-04-29`).
> Generated by an epistemic-deconstructor STANDARD-tier pass.
> Session: `analyses/analysis_2026-04-29_055b8c91/`.

## Executive summary

deepagents is a Claude-Code-style harness library built on LangGraph. Its core idiom is a fixed
middleware stack assembled around a `CompiledStateGraph`, plus declarative profiles that override
per-model behavior. It is **not architecturally aligned with fsm_llm** — fsm_llm runs a typed-λ
executor with a Theorem-2 cost contract, while deepagents pushes everything through a graph-state
channel — so most of its surface is a poor port. But three of its layers are genuinely valuable
**and orthogonal to our substrate**:

1. **HarnessProfile** (per-`provider:model` declarative overrides for prompts, tool
   descriptions, and middleware) — they ship measured 10–20 point lifts on a curated tau2-bench
   subset (CHANGELOG 0.5.4: GPT-5.3 Codex 33% → 53%, Claude Opus 4.7 43% → 53%). This is the
   single highest-value pattern.
2. **Backend / Sandbox protocol** with a `BaseSandbox` whose entire file-ops surface is
   derived from one abstract `execute()` — the right abstraction for the agents-stdlib gap
   we currently paper over with bespoke tools.
3. **Threat-model + deprecation machinery** — small, self-contained, easy to adopt, and they
   raise the floor for production users.

Everything else either duplicates what we already have, embeds LangGraph internals we cannot
inherit, or silently weakens our security story (e.g. memory middleware injecting unsanitized
content into the system prompt).

## Per-pattern verdicts

### 1. HarnessProfile / ProviderProfile — **ABSORB (translate to runtime)**

**What it is.** `deepagents/profiles/__init__.py` exposes `HarnessProfile` and `ProviderProfile`,
keyed by provider (`"openai"`) or full spec (`"openai:gpt-5.4"`), with additive registration
(`register_harness_profile` / `register_provider_profile`). Each `HarnessProfile` carries:

- `base_system_prompt` (replaces SDK default if set), `system_prompt_suffix` (always last);
- `tool_description_overrides: dict[name → description]`;
- `extra_middleware`, `excluded_middleware`, `excluded_tools`;
- `general_purpose_subagent` config.

Composition order (`graph.py:413-430`): `USER → (BASE | CUSTOM) → SUFFIX`, joined with blank
lines. `USER` is always first (caller wins), `SUFFIX` is always last (closest to conversation
where the model attends most).

**Why it matters for fsm_llm.** fsm_llm has **one** Oracle and **one** set of stdlib factories.
Today, prompt and Leaf-description quirks for OpenAI vs Anthropic vs local models are baked into
the factories themselves — invisible heterogeneity. deepagents shows the right factoring:

- Provider-level hooks (analogous to our `LiteLLMOracle._invoke_structured`) become a
  `ProviderProfile` lookup.
- Model-level hooks (system-prompt suffix, Leaf description overrides, handler-stack edits)
  become a `HarnessProfile` keyed on `provider:model`.

**Concrete proposal.**

```
src/fsm_llm/runtime/profiles.py        # new module: HarnessProfile, ProviderProfile, register_*
src/fsm_llm/runtime/_tools.py          # new: _apply_leaf_description_overrides (non-mutating)
src/fsm_llm/program.py                 # apply profile before compile_fsm / compile_term
src/fsm_llm/runtime/_litellm.py        # apply ProviderProfile in resolve_model
docs/profiles.md                       # new doc; cross-link from api_reference.md
tests/test_fsm_llm_lam/test_profiles.py
```

Layered-API placement: **L2 COMPOSE** (profiles are pure AST→AST transforms applied once at
Program construction, never per-turn). They preserve Theorem-2 because they only edit Leaf
descriptions and prompt strings — the executor reduction semantics and `predicted_calls` count
are untouched.

Cost: 250–400 LOC + tests. Risk 2 (touches `__all__` layering). Value 3.

**Critical guardrail.** Apply profile rewrites **once at construction**, exactly like
deepagents `_apply_tool_description_overrides` (`_tools.py`). Re-rewriting per turn would
invalidate the `LiteLLMOracle` prompt cache and break determinism.

### 2. Excluded-component coverage audit — **ABSORB (small)**

`deepagents/_excluded_middleware.py` has three patterns worth lifting independently of #1:

- **Class-or-name exclusion grammar** — accept either `MyMiddleware` (class) or `"MyMiddleware"`
  (string) in the exclusion list.
- **Coverage check** — after assembly, raise `ValueError` if any exclusion matched nothing
  across stacks. Catches typos and stale profiles.
- **Scaffolding protection** — `_REQUIRED_MIDDLEWARE_CLASSES` cannot be excluded, ever.
  Removing them would silently degrade the agent.

For fsm_llm the analogue is the **handler stack** (`HandlerSystem` / `HandlerBuilder`) and
the future profile-driven middleware. Lifting this pattern means: when a profile says
`exclude_handlers=["X"]`, raise loudly if X isn't in the stack.

Lives in `src/fsm_llm/handlers.py` (extension) or `src/fsm_llm/runtime/_excluded.py` (new).
~100 LOC.

### 3. BackendProtocol / SandboxBackendProtocol / BaseSandbox — **TRANSLATE**

**What it is.** `backends/protocol.py` defines:

- `BackendProtocol` — `ls/read/write/edit/grep/glob/upload_files/download_files` with
  structured `*Result` dataclasses carrying `error: str | None` plus payload.
- `SandboxBackendProtocol(BackendProtocol)` — adds `execute(command, *, timeout=None)` plus
  `id` property.
- `BaseSandbox` (`backends/sandbox.py`) — concrete subclass that derives ALL file ops from
  the single abstract `execute()`. Heredoc-fed Python scripts with base64-encoded params
  (THREAT_MODEL D3 dismisses shell-injection threat for this reason). Edit splits at
  `_EDIT_INLINE_MAX_BYTES = 50_000` between inline server-side replace and "upload-as-tempfile,
  replace, cleanup" (source file never leaves the sandbox).

**Why it matters for fsm_llm.** Our `stdlib/agents/` has factories that need to invoke shell,
read files, search, etc. — but we have **no protocol** unifying that surface. Today each agent
factory fashions its own tool callables. The deepagents shape is the right abstraction.

**Why TRANSLATE not ABSORB.** Two reasons:

1. **Stdlib purity rule.** `src/fsm_llm/stdlib/<pkg>/lam_factories.py` imports only from
   `fsm_llm.runtime`. A `LeafBackend` interface must therefore live in `fsm_llm.runtime` or
   be a peer of stdlib. Recommend: `src/fsm_llm/stdlib/agents/backends/` as a new sibling of
   `lam_factories.py`, with the interface re-exported from a runtime-side stub if needed.
2. **Different boundary semantics.** deepagents backends mutate `CompiledStateGraph` channel
   state. fsm_llm needs the protocol at the **Leaf boundary** — a `Leaf` whose
   `output_schema_ref` declares a backend operation, executed by the host-callable fallback
   (the residual case noted in CLAUDE.md post-A.M3c). This keeps Theorem-2 honest:
   non-cohort terminal Leaves are exactly where backend-shaped operations belong.

**Concrete proposal.**

```
src/fsm_llm/stdlib/agents/backends/__init__.py
src/fsm_llm/stdlib/agents/backends/protocol.py   # LeafBackend + SandboxLeafBackend
src/fsm_llm/stdlib/agents/backends/state.py      # ephemeral, in-WorkingMemory
src/fsm_llm/stdlib/agents/backends/local_shell.py
src/fsm_llm/stdlib/agents/backends/sandbox.py    # BaseSandbox-like, exec-derived
docs/agents_backends.md
THREAT_MODEL.md update                           # add T-S* threats per backend
```

Cost 400–600 LOC + 250–400 for sandbox. Risk 3 because a `LocalShellBackend` introduces a new
trust boundary. **Do not ship `LocalShellBackend` without the permissions middleware (#5).**

### 4. CompositeBackend path-prefix routing — **TRANSLATE (only with #3)**

`backends/composite.py` routes by longest-prefix path; aggregates `ls("/")` across all routed
backends. Pattern: `default=StateBackend()`, `routes={"/memories/": StoreBackend()}`. Useful
for "ephemeral by default, persistent for declared paths". Implement only if #3 ships.

### 5. Permissions / first-match-wins rule list — **ABSORB (with #3)**

`middleware/permissions.py` + `FilesystemPermission`:

- Rules evaluated in declaration order; first match wins; default-allow if no match.
- Rules MUST start with `/`; path traversal rejected at construction time (issue #2665).
- Permission middleware appended **last** so it sees all tool-injecting middleware output.

The "always last" placement is a load-bearing safety property — replicate it.

### 6. Threat model artifact (THREAT_MODEL.md) — **ABSORB (write once)**

deepagents ships `libs/deepagents/THREAT_MODEL.md`: scope (in/out), assumptions,
data-classification table, components, trust boundaries, data flows, threats with severity +
validation column ("Verified" / "Likely" / "Disproven"), out-of-scope-threats with rationale,
investigated-and-dismissed table, revision history.

fsm_llm has nothing comparable. Given we ship `LiteLLMInterface` (network-side trust),
`HandlerSystem` (host-side execution), `FileSessionStore` (atomic-rename concurrency), and
plan to add backend-style execution per #3, we need this. Action: write
`docs/threat_model.md` modeled on deepagents'. ~500–700 lines markdown. No code. Posterior 0.71.

### 7. Deprecation runtime machinery — **ABSORB (small, easy)**

`deepagents/_api/deprecation.py` is 90 LOC and ships three things we currently lack:

- `warn_deprecated(..., stacklevel=N)` wrapper that re-emits langchain-core's warning with
  caller-controlled stack attribution. Upstream hardcodes `stacklevel=4`, which mis-attributes
  warnings emitted from method bodies.
- `reset_deprecation_dedupe(*targets)` — pokes the closure-bound `warned` flag in the
  `@deprecated` decorator. **Hack but useful** under `pytest -n auto` where dedupe makes
  per-call assertions reorder-sensitive.
- The `pyproject.toml` `filterwarnings` list with one comment per entry explaining *why*
  that deprecation is silenced in tests — reads as a deprecation log.

fsm_llm has a deprecation calendar in `docs/lambda_fsm_merge.md` §3 but **no runtime
machinery** to enforce or test it. Lift directly. Lives at:

```
src/fsm_llm/_api/__init__.py            # new private package
src/fsm_llm/_api/deprecation.py
pyproject.toml                          # add filterwarnings entries with rationale comments
```

Cost 60–120 LOC. Risk 1. Value 2.

### 8. Per-model eval scorecards — **TRANSLATE**

deepagents CHANGELOG quotes per-`(model × profile)` benchmark numbers. fsm_llm has scorecards
under `evaluation/bench_long_context_*.json` recording `theorem2_holds` per cell, but no
per-model **quality** variance. Recommend extending the schema to include
`profile_id` and `quality_score` columns once #1 ships, then publish a row per
`(model × factory × profile)` cell.

This is what makes a "10–20 point lift" claim auditable instead of marketing.

### 9. Tool description override + non-mutating copy — **ABSORB (helper-sized)**

`deepagents/_tools.py:_apply_tool_description_overrides`: copies the caller's tool list,
rewrites only descriptions on `BaseTool` and dict tools, leaves plain callables alone (you
can't safely replace the description of a callable without wrapping). For fsm_llm this maps
to `Leaf.description` overrides applied via `Term.model_copy(update=...)` (Pydantic frozen).
~50–80 LOC, ships with #1.

### 10. Subagent declarative spec — **DISCARD (mostly)**

`SubAgent` (TypedDict), `CompiledSubAgent` (pre-built runnable), `AsyncSubAgent` (remote
LangGraph deployment), all dispatched through one `task` tool with `subagent_type` allowlist
(`middleware/subagents.py:_build_task_tool`).

In fsm_llm:

- `SubAgent` declarative is **redundant** with `react_term` / `rewoo_term` / `reflexion_term`
  — those *are* the typed-AST equivalents. We don't need a TypedDict shim.
- `CompiledSubAgent` is **redundant** with `Program.from_term`.
- `AsyncSubAgent` is **LangGraph-Cloud-only** and out of scope.

**Worth lifting separately**: the auto-injected "general-purpose" subagent (`graph.py` adds
one unless the caller supplies it or a profile disables it) — it solves a real UX problem
("what runs when no specific path matches?"). Translate as an optional `default_factory=`
parameter on `Program.from_factory`. Half a dozen lines.

### 11. Skills middleware (SKILL.md + YAML frontmatter) — **DISCARD**

`middleware/skills.py` parses SKILL.md frontmatter via `yaml.safe_load`, caps file size at
10MB, injects metadata into the system prompt, loads the body on demand. Validation: name
length 64, description length 1024 — but **invalid names only warn, they don't reject**
(`middleware/skills.py:302-308`).

For fsm_llm this overlaps `prompts.py` and provides no new mechanism. The "load fragment by
reference" pattern is real, but the right shape in our world is a typed `Leaf` whose
`output_schema_ref` is a fragment id — not a middleware mutating the system prompt.

### 12. Memory middleware (AGENTS.md) — **DISCARD with caveat**

`middleware/memory.py` reads AGENTS.md, decodes UTF-8, interpolates via `str.format()` into
`MEMORY_SYSTEM_PROMPT`. **No size cap, no sanitization.** THREAT_MODEL T1 lists this as
*Verified High*: a poisoned `.deepagents/AGENTS.md` in a cloned repo can inject arbitrary
instructions into the system prompt, and the surrounding prompt actively instructs the model
to treat memory as authoritative.

We already have `WorkingMemory` (4 named buffers) and `ContextCompactor` (transient-key
clearing). If we want runtime-loaded memory, do it **as a typed Leaf with size+content
validation**, not as middleware that string-formats unvalidated bytes into the system prompt.

### 13. PatchToolCallsMiddleware — **DISCARD**

Model-quirk plumbing for malformed tool-call JSON. `LiteLLMOracle._invoke_structured`
already abstracts provider quirks; adding a middleware layer would duplicate.

### 14. StateBackend internals — **DISCARD**

`backends/state.py` is built on `CONFIG_KEY_READ` / `CONFIG_KEY_SEND` Pregel internals,
fresh-read for read-your-writes within superstep. Pure LangGraph coupling. The corresponding
fsm_llm primitive is `WorkingMemory` + `Session`; we already have it.

### 15. `_apply_profile_prompt`, prompt assembly — **ABSORB (with #1)**

The `USER → (BASE | CUSTOM) → SUFFIX` order from `graph.py:413-430`, with `SystemMessage`
content-block preservation for `cache_control` markers, is the right shape and can be ported
verbatim into the fsm_llm profile module.

### 16. CRLF / path / sandbox craftsmanship — **NOTE**

Worth bookmarking but not absorbing as a pattern:

- Match-driven CRLF handling in `_EDIT_COMMAND_TEMPLATE` (try `old` as-sent, then CRLF,
  then LF; first match wins; same transform applied to `new`). Issue #2880.
- Boundary-truncated UTF-8 prefix check in `read()` uses an incremental decoder so multi-byte
  chars at the 8192-byte cut aren't misclassified as binary (#2980).
- Atomic `os.O_NOFOLLOW` flag on every `open` to avoid symlink writes.

If we ship #3 (sandbox), inherit these.

Theorem-2 or the layered `__all__`.

## What we explicitly do NOT take, and why

1. **LangGraph middleware idiom** — we have a typed executor with handlers; replacing it with
   `modify_request` / `before_model` hooks would dissolve our core thesis.
2. **`task` tool + Subagent TypedDict spec** — our stdlib factories ARE the subagent layer at
   the right abstraction level.
3. **Skills/Memory as middleware** — security risk + overlaps existing memory primitives.
   Would weaken our trust story without paying for itself.
4. **AsyncSubAgent / LangGraph-Cloud integration** — out of thesis.
5. **PatchToolCallsMiddleware** — duplicates `LiteLLMOracle` provider abstraction.
6. **deepagents' choice to default `LocalShellBackend(virtual_mode=False)`** — when we ship a
   sandbox, default to virtual mode and require explicit opt-out.
